# MegaResearcher v1 — Phase 1: Fork & Trim claurst Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring claurst's Rust workspace into this repo as the base for MegaResearcher v1, trim the non-research crates (`buddy`, `acp`, `bridge`) and the voice/mic + `/share` features, add a `research` crate stub, set the GPL-3.0 license, and leave the workspace building, linting, testing, and formatting clean.

**Architecture:** claurst is a Cargo workspace under `src-rust/` with 12 member crates. We copy the 9 we keep (`core`, `api`, `tools`, `query`, `tui`, `commands`, `mcp`, `plugins`, `cli`) into a new root-level `crates/` workspace in this repo, omit the 3 we delete (`buddy`, `acp`, `bridge`), fix the two crates that referenced the deleted ones (`cli`, `commands`), remove the `voice` feature, add a `research` crate stub, and verify the whole workspace is green. The existing Apache-2.0 plugin files at repo root coexist untouched until the Phase 9 cutover.

**Tech Stack:** Rust 2021 edition, Cargo workspace (resolver 2), the dependency set already pinned in claurst's `Cargo.toml` (tokio, reqwest/wreq, ratatui, crossterm, rmcp, serde, clap, etc.). GPL-3.0 license.

## Global Constraints

- **No git worktrees.** Work directly in the repo's main checkout on the current branch (`main`). Do not `git switch` mid-task. (Hard rule from the user's global instructions.)
- **License:** The harness is GPL-3.0. Root `LICENSE` = GPL-3.0 full text. `NOTICE` documents that `assets/prompts/**` (added in a later phase) remains Apache-2.0.
- **Old plugin files stay.** Do not delete `agents/`, `skills/`, `lib/`, `mcp/`, `tools/`, `commands/`, `hooks/`, `.claude-plugin/` — they are the port reference and are removed only in Phase 9. Cargo ignores them.
- **Verify from repo root**, not from `src-rust/` (claurst ran inside `src-rust/`; our workspace root is the repo root).
- **Banned words:** never use "load-bearing" or "this is doing a lot of work" in any artifact. (User global rule.)
- **Binary name:** keep claurst's `claurst` binary name for this phase. The rename to `mr` happens in Phase 6 (interface tailoring). Do not rename now — it touches clap wiring and is out of scope here.
- **Out of scope for this plan (deferred):** collapsing claurst's multi-provider "Free Mode" routing and dropping `wreq`/`wreq-util` (touches `api/src/providers/`, `registry.rs`, `model_registry.rs` — a meatier source edit done when we tailor the provider layer); trimming the ~30 experimental core/tui pass-through feature flags; renaming crates to `megaresearcher-*`; the `mr` binary rename. These are listed in the spec's later phases.
